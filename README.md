# Curio

Curio 是一个语音优先的轻量灵感捕捉 Web 验证版。

当前版本用于验证核心体验：打开后直接说出想法，必要时简单修订，然后收好；之后可以在灵感列表里搜索、喜欢、编辑、删除，也可以用“翻翻”随机回看旧想法。

## 如何打开

直接打开 `index.html` 即可预览。

如果想用本地服务打开：

```bash
python3 -m http.server 4173
```

然后访问：

```text
http://localhost:4173
```

## 当前功能

- 语音优先输入：点击主图开始听写；不支持原生听写的浏览器会聚焦输入框，可用系统键盘麦克风。
- 文本修订：识别后可以补充或修改。
- 本地保存：数据保存在浏览器 `localStorage`。
- 灵感列表：支持搜索、喜欢、编辑、删除。
- 翻翻：随机回看旧灵感。
- 样例数据：首次打开且没有本地数据时，会展示 7 条产品方向样例灵感。
- PWA 预览：通过本地服务访问时，可验证主屏幕入口和基础离线缓存。

## 当前限制

- 没有账号、云同步或多设备备份。
- 原生语音识别依赖浏览器支持和麦克风权限；微信等内置浏览器可能需要使用系统键盘麦克风。
- PWA 只是验证版外壳，不代表已经进入正式 App 发布阶段。
- Markdown 导出、Obsidian 连接、AI Curator's Note 暂时只在数据结构里预留。

## 部署

Curio 当前是纯静态 Web 验证版，可以直接部署到 GitHub Pages、Vercel 或 Netlify。

GitHub Pages：

1. 把仓库推到 GitHub。
2. 在仓库 `Settings -> Pages` 里选择 `main` 分支和根目录。
3. 等待 Pages 生成 HTTPS 链接。

Vercel：

1. 导入这个仓库。
2. Framework Preset 选择 `Other`。
3. Build Command 留空，Output Directory 留空或使用根目录。

部署后重点用手机验证：语音权限、`localStorage` 保存、刷新保留、主屏幕入口和 PWA 图标。

## 验证重点

- 打开后是否愿意直接说出灵感。
- 说完后修订和保存是否自然。
- 灵感列表是否让人愿意回来翻。
- “翻翻”是否有重新发现旧想法的感觉。
